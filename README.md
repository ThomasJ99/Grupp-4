## 🤖 3. AI-Policy & Kodkultur
Hur vi använder moderna verktyg och säkerställer kvalitet:

* **AI-användning:**
    * [ ] **Rådgivande:** Vi använder AI som ett bollplank för logik och felsökning, men skriver koden manuellt.
    * [ ] **Generativ (Vibe-coding):** Vi får använda AI för att generera kodblock om vi vill men håller det rådgivande, men den som checkar in koden ansvarar för att kunna förklara          exakt vad den gör för resten av gruppen.
    Gå igenom koden om man känner sig osäker vad den gör tillsammans med andra och refaktorera den.
* **Code Reviews:** Innan en Pull Request (PR) mergas till `main` / `dev` ska minst **1** annan teammedlem granska koden. Den som granskar bör testa koden lokalt på VSC vid större PR's.
* **Kunskapsdelning:** Om någon fastnar i mer än **60** minuter ber man om hjälp eller föreslår en kort session med parprogrammering.

## 🌿 4. Git-strategi & Versionshantering
För att undvika merge-konflikter och trasig kod:

* **Branching:** Vi skapar nya branches från `dev`. Namngivning: **feature/issue**.
   * Branches
       bugfix/issue
           Use bugfix for minor fixes to styling or other code snippets
       Work in components and try to limit code to only said component
  
* **Merges:** Vi gör aldrig en "Force Push" till main. Om konflikter uppstår löser vi dem tillsammans i gruppen vid behov.
* **Frekvens:** Vi pushar vår kod ofta (minst en gång per dag) för att undvika stora, svårhanterliga PRs.
* **MVP:** Varje fredag/måndag gör vi en push till main med nuvarande features så håller vi vår main clean och fungerande.

## 💬 5. Kommunikation & Konflikthantering
Vi lovar att vara professionella och schyssta mot varandra:

* **Feedback:** Vi ger konstruktiv feedback på kod.
* **Eskaleringsplan (om någon går "AWOL"):**

    2.  **Slutet av Onsdag - 16:30 :** Kort avstämning i gruppen: "Hur mår vi? Behöver vi omfördela uppgifter? Kolla läget.".
    3.  **3 timmar utan livstecken:** Vi kontaktar handledare/lärare gemensamt för att få hjälp att lösa situationen.
* **Beslut:** Vi strävar efter konsensus. Vid oenighet röstar vi, och vid dött lopp rådfrågar vi läraren.
